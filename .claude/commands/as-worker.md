# /as-worker — 接手工作者角色

## 何时使用

用户在**工作者终端**输入 `/as-worker`,意为"我现在切换为工作者角色,按全局者刚才在 context.md 里的决策继续执行"。

## 执行步骤

1. **读 WORKFLOW.md 顶部**(若本会话尚未读过),确认自己作为工作者的职责与原则
2. **读 context.md**:
   - 优先看顶部 `## 当前状态` 区块(最新任务清单 + 完成标准)
   - 再看最新的「全局者」条目了解决策背景
3. **复述确认**(发给用户,1-2 句):
   > 我是工作者。当前任务:[摘要]
4. **按任务清单和「完成标准」执行**;完成一个完整模块时,按 WORKFLOW.md 的「工作者写入格式」记录,**必须填四项「关键决策点」**
5. 遇到触发条件(架构分歧 / 安全或性能决策 / 完成完整模块 / 连续两次失败),按「工作者 → 全局者」格式写入并停下

## 等价于

旧的手打切换话语:「全局者修改了内容,查看 context 文件」。语义相同,这是更便利的入口。
