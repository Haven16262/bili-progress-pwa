# 双模型工作流

## 角色定义

### 全局者
负责方向、决策、审查。每次交互必须产出明确结论并写入 context.md。

职责：
- 项目启动时制定架构和方向
- 工作者遇到分歧时给出决策
- 阶段性审查工作者输出，确认方向后放行
- 最终 review

原则：**不做细节执行，不读大量文件，不做迭代修改。**

---

### 工作者
负责执行、迭代、细节实现。默认运行角色。

职责：
- 读取 context.md 获取当前方向和任务
- 执行具体实现：读文件、查资料、写代码、迭代修改
- 完成阶段后将结果摘要追加写入 context.md

原则：**遇到触发条件时立即停下，告知用户切换到全局者。**

---

## 切换触发条件

### 工作者 → 全局者
1. 出现架构方向分歧，有多个方案无法判断选哪个
2. 遇到安全、性能相关的设计决策
3. 完成一个完整模块，需要确认方向再继续
4. 连续两次尝试仍无法解决某个问题

触发时**必须按顺序执行**：
1. 按以下格式将交接内容追加写入 context.md：
```
## [YYYY-MM-DD HH:MM] 工作者 → 全局者
**问题：** [遇到了什么]
**已尝试：** [做了什么、为什么不行]
**需要决策：** [全局者需要回答什么]
```
2. 再说：**「[触发原因]，已写入 context.md，请切换到全局者处理。」**

### 全局者 → 工作者
1. 决策已写入 context.md，可以继续执行
2. 审查通过，下一阶段方向明确

触发时**必须按顺序执行**：
1. 先将决策写入 context.md
2. 再说：**「决策已写入 context.md，可切换回工作者继续。」**

---

## context.md 写入规范

每次写入追加到文件末尾。

### 全局者写入格式

写入时**同时做两件事**：
1. 覆盖更新 context.md 顶部的 `## 当前状态` 区块（用最新任务清单替换旧内容）
2. 追加历史记录到文件末尾：

```
## [YYYY-MM-DD HH:MM] 全局者
**背景：** [为什么做这件事]
**决策：** [具体决策内容]
**任务清单：**
- [ ] 任务 1
- [ ] 任务 2
- [ ] 任务 3
```

纯方向确认（无需任务分解）时，省略任务清单只写背景和决策即可。

### 工作者写入格式

```
## [YYYY-MM-DD HH:MM] 工作者
**完成情况：** [阶段结果摘要]
**任务进度：**
- [x] 任务 1
- [x] 任务 2
- [ ] 任务 3（遗留：[原因]）
```
